Tuples — The Locked Treasure Chest

You already know lists. A list is like a treasure chest you can always open and change:

LIST (you know this!)
chest = ["gold", "silver", "ruby"]
chest[0] = "diamond"  # fine! we can change it

A tuple is a chest with a magic lock. Once you seal it, nobody can change it. Not even you!

Imagine you've already delivered treasure to the Queen. You can't swap items in a delivered chest — the deal is done! That's a tuple: permanent, official, sealed.
TUPLE — sealed with ( ) instead of [ ]
royal_chest = ("gold", "silver", "ruby")

# Reading works the same as lists:
print(royal_chest[0])   # → gold
print(royal_chest[2])   # → ruby

# But this would EXPLODE 💥:
# royal_chest[0] = "diamond"  ← TypeError!

Why use tuples? When data should never change — like coordinates, dates, or a pirate's true name.

REAL EXAMPLE
island_location = (14.5, -75.3)  # latitude, longitude
pirate_birthday = (1999, 7, 14)    # year, month, day

# You can unpack tuples!
year, month, day = pirate_birthday
print(year)   # → 1999

map() — The Magic Transformation Spell

Imagine you have a list of cannonball weights in pounds, and you need them in kilograms. You could write a for-loop…

THE OLD WAY (for loop)
pounds = [10, 20, 30, 50]
kilos = []
for p in pounds:
    kilos.append(p * 0.453)
print(kilos)
map() is like hiring a spell-caster on your ship. You say:
"Take EVERY item in this barrel, cast this spell on it, and give me a new barrel."
You don't write the loop — the spell-caster does it for you!
THE map() WAY ✨
pounds = [10, 20, 30, 50]

def to_kilos(p):
    return p * 0.453

kilos = list(map(to_kilos, pounds))
print(kilos)
# → [4.53, 9.06, 13.59, 22.65]

⚠️ Important: map() returns a lazy "map object" — wrap it in list() to see the results!

map() RECIPE — always the same shape
# list( map( FUNCTION, YOUR_LIST ) )
result = list( map( my_function, my_list ) )

# Works on strings too!
names = ["anne", "blackbeard", "jack"]
shouting = list(map(str.upper, names))
print(shouting)
# → ['ANNE', 'BLACKBEARD', 'JACK']

zip() — The Pairing Rope

You have two separate lists: pirates and their ships. You want to connect them — pair each pirate with their ship.

zip() is like tying two ropes together, knot by knot.
Rope 1: ["Anne", "Jack", "Mary"]
Rope 2: ["Queen Anne", "Revenge", "Flying Fish"]
zip knots them: Anne↔Queen Anne, Jack↔Revenge, Mary↔Flying Fish
zip() BASIC EXAMPLE
pirates = ["Anne", "Jack", "Mary"]
ships   = ["Queen Anne", "Revenge", "Flying Fish"]

pairs = list(zip(pirates, ships))
print(pairs)
# → [('Anne', 'Queen Anne'), ('Jack', 'Revenge'), ('Mary', 'Flying Fish')]

Notice: each pair is a tuple! zip() creates tuples automatically — those sealed chests from Island 1!

LOOP OVER zip() — super useful!
pirates = ["Anne", "Jack", "Mary"]
coins   = [500, 200, 750]

for pirate, gold in zip(pirates, coins):
    print(pirate, "has", gold, "gold coins")

# → Anne has 500 gold coins
# → Jack has 200 gold coins
# → Mary has 750 gold coins

💡 What if lists have different lengths? zip() stops at the shorter list — it won't crash, just stops early.

Your Cheat Sheet — The Pirate's Codex

You've sailed all three islands! Here's everything in one place:

TOOL WHAT IT DOES HOW TO USE IT
tuple Sealed list — can't be changed x = (1, 2, 3)
unpack Split a tuple into variables a, b, c = (1, 2, 3)
map() Apply a function to every item list(map(fn, lst))
zip() Pair up two lists, item by item list(zip(lst1, lst2))
zip in for Loop over two lists at once for a, b in zip(x, y):

Now use all three tools together — this is captain-level Python!

ALL THREE TOOLS TOGETHER 🏴‍☠️
# A pirate shop with items and prices
items  = ["sword", "map", "compass"]
prices = (150, 30, 80)  # tuple — prices never change!

def add_tax(p):
    return p * 1.2   # 20% pirate tax

taxed = list(map(add_tax, prices))   # map!

for item, price in zip(items, taxed):  # zip!
    print(f"{item}: {price:.1f} gold")

# → sword: 180.0 gold
# → map: 36.0 gold
# → compass: 96.0 gold
